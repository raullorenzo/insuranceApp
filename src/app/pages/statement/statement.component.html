What we want you to do

As a user I want to have access to an Insurance Manager with a search feature providing the following criteria:

  • Product name
  • Insurance Company (i.e.: Zurich, AXA, Liberty, etc, etc,...)
  • Policy Kind (Car, House, Life, etc, etc...)
  • Price

Each policy will have these 4 fields.

When I perform a search which has results, I’d like to see its resulting insurance list composed by those 4 fields and the company logo. I also want to have the ability to sort the list by company, kind and price with the option to decide if it is in ascending or descending order.

I also want to be able to manage a favourite policies list, so I want to be able to select policies from the result list and save them on my favourite list. Favourites should be displayed on a modal as a list similar to the search result list but without the price. I would like to be able to open this modal by clicking on a button, which should be placed on the page in a manner that it’s always easily accessible.

In the favourites modal, I want to be able to search by name, company and kind and have the possibility to remove the policies from the favourite list, without having to close the modal.

The JSON with the policies information contains 33 entries, but they should NOT be displayed all at once. Lists should be built with 5 policies at time, with an initial load of 5 when the page is loaded, so some pagination feature should be implemented in order to view the remaining items (clicking a button, with endless scroll...it’s up to you).

Delivery requirements

You can use any strategy you prefer for storing the favourites list and it is not required that such list has to persist after page reload.

You must attach your solution with all the files required for making it work in a single zip file or provide the same files by a BitBucket or a GitHub private repository.

The solution must run properly and a text file with any instruction required for compiling and/or running it should be provided as it would be directed to a no technical user.
